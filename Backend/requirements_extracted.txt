Requirements Analysis
This section presents the requirements analysis of the system “Beauty Personalized by AI”. It identifies system actors, defines the product backlog, and outlines all functional and non-functional requirements.

3.2 Product Backlog
The product backlog contains all system features described as user stories. Each item includes priority, acceptance criteria, and story points.
3.2.0 Agile Concepts
This project follows the Agile methodology for software development. The following concepts are used to manage and organize the product backlog.
Link for WBS Diagram:

https://miro.com/app/board/uXjVGd-LPz8=/
Sprint
A sprint is a fixed time period during which the development team completes a set of tasks and delivers a working product increment.
Sprints typically last 2 to 3 weeks in this project. Each sprint includes planning, development, testing, and review.
Epic
An epic is a large user story that cannot be completed in a single sprint. It represents a major functionality of the system.
Epics are divided into smaller user stories for implementation.
Example: "Implement booking system"
Feature
A feature is a specific functionality within an epic that provides value to the user.
Features are smaller than epics and can be completed within a sprint.
Example: "Select appointment time" "Online payment"
User Story
A user story is a short description of a system feature from the user's perspective.
Format:
As a , I want so that 
Example: As a customer, I want to book an appointment so that I can visit the salon easily.
Acceptance Criteria
Acceptance criteria define the conditions that must be met for a user story to be considered complete.
They describe the expected behavior of the system.
Example:
User Story: As a user, I want to reset my password so that I can access my account.
Acceptance Criteria:
User can enter email to request reset
System sends reset link
Link expires in 24 hours
User can set new password
Story Points
Story points are used to estimate the effort required to complete a user story.
They represent complexity, effort, and risk, not time.
Common scale: 1, 2, 3, 5, 8, 13
2–3 → Simple task
5 → Medium task
8 → Complex task
Backlog Grooming
Backlog grooming is the process of reviewing and updating the product backlog regularly.
It includes:
Refining user stories
Breaking large tasks into smaller ones
Estimating effort
Prioritizing tasks
Adding or removing backlog items
Importance
Improves sprint planning
Increases team productivity
Keeps backlog up to date
Reduces project risks
